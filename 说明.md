# BFireworks 前端结构说明

## 目录

```
Front-end/
├── index.html               ← 入口文件，只有 HTML 骨架 + 引用 JS/CSS
├── css/
│   └── style.css            ← 全部样式（约 370 行）
├── js/
│   ├── api.js               ← 工具函数：api请求、toast提示、esc转义、icon图标
│   ├── auth.js              ← 登录/注册/登出、会话保持
│   ├── sidebar.js           ← 侧边栏菜单定义、渲染、导航跳转、折叠
│   ├── app.js               ← 页面路由器 render()、首页home、烟花页、设置页
│   └── pages/
│       ├── announcements.js ← 公告列表、发布、编辑、删除
│       ├── users.js         ← 用户管理CRUD、列表、搜索
│       └── fireworks.js     ← 烟花引擎、登录页背景烟花、鼠标拖尾
└── img/
    └── login-bg.jpg         ← 登录页背景图
```

## 加载顺序

浏览器打开 `index.html` 后：

```
index.html
  ├─ css/style.css         → 样式生效
  ├─ js/api.js             → 全局工具函数可用
  ├─ js/auth.js            → 登录注册逻辑
  ├─ js/sidebar.js         → 侧边栏渲染
  ├─ js/app.js             → 页面路由 + 首页/设置页
  ├─ js/pages/announcements.js
  ├─ js/pages/users.js
  └─ js/pages/fireworks.js → 最后加载（烟花引擎较大）
```

## 加新功能只需两步

**1. 新建页面文件** `js/pages/xxx.js`，写一个 `pageXxx()` 函数

**2. 在 `js/app.js` 的 `render()` switch 里加一行：**
```javascript
case 'xxx': h = pageXxx(); break;
```

**3. 在 `js/sidebar.js` 的 `menu` 数组里加菜单项：**
```javascript
{ id: 'xxx', icon: 'i-list', label: '新功能' },
```

## 关键全局变量

| 变量 | 说明 |
|------|------|
| `me` | 当前登录用户 `{id, username, role}` |
| `curPage` | 当前页面ID |
| `API` | 后端地址 `http://localhost:8800` |
| `TOKEN` | 登录令牌（存 sessionStorage） |

## 和旧单文件版的区别

- 旧：1235 行一个文件，改一个标点整个文件都红
- 新：每个模块独立文件，改公告不影响烟花，改样式不动 JS
